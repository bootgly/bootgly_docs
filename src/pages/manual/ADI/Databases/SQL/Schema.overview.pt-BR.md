# Schema Builder

O Schema Builder é **como você molda seu banco**. Você o chama de dentro de uma migration
para criar, alterar ou remover tabelas — nunca SQL na mão. Ele escreve o SQL certo para sua
engine (PostgreSQL, MySQL ou SQLite) por você.

> Novo aqui? Siga primeiro o guia
> **[Database migrations](/guide/database-migrations/overview/)** — ele mostra o fluxo
> inteiro. Esta página é o que você faz dentro do `Up`/`Down`.

## As cinco coisas que você vai fazer

Dentro de uma migration você recebe um `$Schema`. Com ele você quase sempre faz uma destas:

```php
// 1. Criar uma tabela
$Schema->create('users', function (Blueprint $Table): void {
   $Table->add('id', Types::BigInteger)->generate()->constrain(Keys::Primary);
   $Table->add('email', Types::String)->limit(190)->constrain(Keys::Unique);
   $Table->add('active', Types::Boolean)->default = true;
   $Table->add('created_at', Types::Timestamp)->default = new Expression('CURRENT_TIMESTAMP');
});

// 2. Alterar uma tabela existente
$Schema->alter('users', function (Blueprint $Table): void {
   $Table->add('bio', Types::Text)->nullable = true;  // nova coluna anulável
   $Table->remove('legacy');                           // remove uma coluna
});

// 3. Remover uma tabela
$Schema->drop('users');

// 4. Renomear uma tabela
$Schema->rename('users', 'members');

// 5. Criar / remover um índice
$Schema->index('users', ['email', 'created_at'], unique: true);
$Schema->unindex('users', 'users_email_created_at_unique');
```

> `nullable` e `default` são **propriedades**, não métodos: você as atribui
> (`->default = true`, `->nullable = true`), não as encadeia. Para remover um default
> existente em `alter`, atribua `Defaults::None`. Detalhes em
> **[Definindo tabelas](/manual/ADI/Databases/SQL/Schema/Blueprint/overview/)**.

**O que isso produz?** Cada chamada retorna uma query compilada — não roda até a migration
aplicá-la. Para o primeiro `create` acima, no PostgreSQL você obtém:

```sql
CREATE TABLE "users" ("id" BIGINT GENERATED BY DEFAULT AS IDENTITY NOT NULL PRIMARY KEY, "email" VARCHAR(190) NOT NULL UNIQUE, "active" BOOLEAN NOT NULL DEFAULT TRUE, "created_at" TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP)
```

Como descrever colunas (a parte `$Table->add(...)`) está em
**[Definindo tabelas](/manual/ADI/Databases/SQL/Schema/Blueprint/overview/)**.

## Bom saber

- **Seguro por padrão.** `drop`, `unindex` e `unconstrain` usam `IF EXISTS`, então rodar a
  migration de novo não quebra. Quer estrito? `$Schema->drop('users', exists: false)`.
- **Create idempotente.** `$Schema->create('settings', $build, exists: true)` →
  `CREATE TABLE IF NOT EXISTS …`. Tipo padrão de coluna é `Types::Text`.
- **Tabela vazia é rejeitada.** Um `create` sem colunas lança `InvalidArgumentException`.
- **Nomes de índice são automáticos.** `index()` nomeia como
  `<tabela>_<colunas>_<index|unique>` e trunca ao limite de identificador da engine
  (63 chars no PostgreSQL).

## Referência

Obtenha um `$Schema` fora de uma migration via banco. A engine ativa está em
`$Schema->Dialect`.

```php
use Bootgly\ADI\Databases\SQL;

$Schema = new SQL->structure();                         // PostgreSQL
$Schema = new SQL(['driver' => 'mysql'])->structure();  // MySQL
$Schema = new SQL(['driver' => 'sqlite'])->structure(); // SQLite
```

Nomes aceitam `string`, qualquer `Stringable` ou um **backed enum** (seu `value` é usado).

### create

```php
create (BackedEnum|Stringable|string $Table, Closure $Build, bool $exists = false): Query
```

Compila um `CREATE TABLE`. A closure recebe um `Blueprint` novo. `exists: true` emite
`CREATE TABLE IF NOT EXISTS`. Blueprint vazio lança `InvalidArgumentException`.

### alter

```php
alter (BackedEnum|Stringable|string $Table, Closure $Build): Query
```

Compila um `ALTER TABLE`. A closure recebe um `Blueprint` novo; cada ação de add/drop/
change/rename/reference é agrupada num único statement quando a engine suporta:

```php
$Schema->alter('users', function (Blueprint $Table): void {
   $Table->add('bio', Types::Text)->nullable = true;
   $Table->remove('legacy');
   $Table->reference('team_id', 'teams', 'id')
      ->delete(References::Cascade)
      ->update(References::Restrict);
});
// ALTER TABLE "users" ADD COLUMN "bio" TEXT, DROP COLUMN "legacy", ADD CONSTRAINT "users_team_id_teams_fk" FOREIGN KEY ("team_id") REFERENCES "teams" ("id") ON DELETE CASCADE ON UPDATE RESTRICT
```

### drop

```php
drop (BackedEnum|Stringable|string $Table, bool $exists = true): Query
```

Compila um `DROP TABLE`. `IF EXISTS` por padrão; passe `exists: false` para estrito.
`$Schema->drop('users')` → `DROP TABLE IF EXISTS "users"`.

### rename

```php
rename (BackedEnum|Stringable|string $From, BackedEnum|Stringable|string $To): Query
```

Compila o rename de tabela. `$Schema->rename('users', 'members')` →
`ALTER TABLE "users" RENAME TO "members"`.

### index

```php
index (BackedEnum|Stringable|string $Table, string|array $Columns, null|string $name = null, bool $unique = false): Query
```

Compila um `CREATE INDEX`. Sem `name`, gera `<tabela>_<colunas>_<index|unique>`, truncado
ao limite de identificador da engine.

```php
$Schema->index('users', ['email', 'created_at'], unique: true);
// CREATE UNIQUE INDEX "users_email_created_at_unique" ON "users" ("email", "created_at")
```

### unindex

```php
unindex (BackedEnum|Stringable|string $Table, BackedEnum|Stringable|string $Name, bool $exists = true): Query
```

Compila um `DROP INDEX` (`IF EXISTS` por padrão). PostgreSQL mantém o qualificador de schema
da tabela: `$Schema->unindex('audit.users', 'users_email_index')` →
`DROP INDEX IF EXISTS "audit"."users_email_index"`.

### unconstrain

```php
unconstrain (BackedEnum|Stringable|string $Table, BackedEnum|Stringable|string $Name, bool $exists = true): Query
```

Compila um `DROP CONSTRAINT` (`IF EXISTS` por padrão).
`$Schema->unconstrain('users', 'users_team_id_teams_fk')` →
`ALTER TABLE "users" DROP CONSTRAINT IF EXISTS "users_team_id_teams_fk"`.

Diferenças de saída por engine: **[Dialects](/manual/ADI/Databases/SQL/Schema/Dialects/overview/)**.
